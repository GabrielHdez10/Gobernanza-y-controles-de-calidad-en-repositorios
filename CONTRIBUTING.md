# CONTRIBUTING.md

## Reglas del equipo

Este repositorio sigue reglas de colaboración para asegurar calidad y orden en todos los cambios antes de integrarlos a la rama principal.

---

## 1. Nombres de archivos

Todos los archivos deben seguir el formato **kebab-case**.

### Regla:
- Solo letras minúsculas
- Usar guiones medios `-`
- No usar espacios
- No usar acentos ni caracteres especiales

### Ejemplos correctos:
- documento-de-arquitectura.md
- organizador-de-archivos.py
- reporte-final.md

### Ejemplos incorrectos:
- Documento Final.md
- organizador_de_archivos.py
- ReporteFinal.md

---

## 2. Estructura obligatoria de documentos Markdown

Todo archivo Markdown debe contener, como mínimo, las siguientes secciones:

1. **Título**
2. **Autor**
3. **Fecha**
4. **Contenido**

```md
# Título del documento

**Autor:** Nombre del alumno
